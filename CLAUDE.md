## Project Overview

Personal academic website for Shujie XU, built with Hugo (no theme dependency -- fully custom layouts). Deployed to GitHub Pages at `https://Shujie-Xu.github.io/` via GitHub Actions on push to `main`.

## Development Commands

```bash
hugo server -D          # Local dev server with drafts (http://localhost:1313)
hugo                    # Build site to ./public
hugo --gc --minify      # Production build (matches CI)
```

Hugo extended edition is required (CI pins v0.146.0; local install is via Homebrew).

## Architecture

Theme-less Hugo site -- all templates live in `layouts/`, not in `themes/`. No external theme submodule despite the `themes/` directory existing.

### Layout chain

- `layouts/_default/baseof.html` -- base shell (head with Google Fonts preconnect + Lora/Newsreader load, header with nav + dark mode toggle, footer). All pages inherit this.
- `layouts/index.html` -- homepage (`{{ define "main" }}`). Reads front matter in `content/_index.md` to render hero, research-focus pull quote, and current-work numbered list (roman numerals via Hugo slice lookup). Hero h1 includes a blinking `.caret` span and the first prose paragraph gets a drop-cap via CSS.
- `layouts/cv/list.html` -- CV page with embedded `<iframe>` PDF viewer. Reads `pdf` and `updated` from front matter.
- `layouts/_default/list.html` / `single.html` -- generic section and single page templates (no eyebrow; the h1 alone heads the page).
- `layouts/partials/social-icons.html` / `svg-icon.html` -- social-link row and inline SVG icon lookup by name.

### Content structure

- `content/_index.md` -- homepage. Front matter drives `intro` (kicker, heading, lead, location), `photo`, `highlights` (Training / Fields / Methods / Perspective), and `current` (list of in-flight projects). Body is bio prose. `intro.lead` can contain `<br>` because `layouts/index.html` renders it with `| safeHTML` -- used to force a line break between sentences.
- `content/research/_index.md` -- uses real heading hierarchy: `##` for sections (Working Papers / Work in Progress / Research Experience), `###` for individual paper or position titles, `*italic*` for metadata lines (dates, coauthors), regular paragraphs for descriptions. Do not revert paper titles back to `**bold**` -- the visual hierarchy depends on the `###` being real `<h3>`.
- `content/publications/_index.md` -- placeholder.
- `content/cv/_index.md` -- CV page frontmatter (`pdf` path under `/files/`, `updated` date string). Body is optional extra prose rendered under the viewer.

### Styling

Single stylesheet: `assets/css/main.css`. CSS custom properties for light/dark theming (`data-theme` attribute on `<html>`).

**Two-font system** (no mono, no third family):
- `--font-display` Lora -- all headings, `.site-logo__name`, `.current-num` (roman numerals), `.footer-name`, `.fit-statement` (italic), drop cap.
- `--font-body` Newsreader (variable, opsz + wght axes) -- body prose, italic labels (`.eyebrow`, `.fit-label`, `.highlights dt`), meta lines, nav.

**Typography hierarchy invariants** (headings must stay heavier than body, else layering collapses visually):
- Body: wght 480 via `font-variation-settings: "opsz" 18, "wght" 480`.
- H1 (hero / page-header): wght 650. H2: wght 650. H3: wght 600. All via `font-weight` property, not via variation-settings, so Lora's wght axis picks it up.
- Italic labels: wght 480, Newsreader italic -- differentiated from body by `font-style`, not weight.
- Drop cap (`.hero-text .prose > p:first-of-type::first-letter`) and `.caret` both use `--accent` (terracotta) -- the two primary accent anchors.

**`font-variation-settings` gotcha**: it is an inherited CSS property that **replaces** (not merges) when re-declared. Any selector that sets `font-variation-settings` loses the inherited body `wght` 480 and falls back to the font's default weight (≈400, often reading as "too thin"). When setting variation-settings on a new rule, re-include `"wght"` explicitly or rely on `font-weight` property.

### Configuration

`hugo.toml`:
- `[params]` -- name, role, location, email, description, footer.text, and `social` array.
- `[menu.main]` -- Research, Publications, CV. CV points to `/cv/` (internal page), not the PDF direct link.
- `params.social` array uses `icon` names resolved by `layouts/partials/svg-icon.html`.
- `[markup.goldmark]` has `unsafe = true` and the passthrough extension enabled for `$...$` / `$$...$$` LaTeX math.

## Deployment

`.github/workflows/deploy.yml` builds with Hugo extended v0.146.0 on push to `main` and deploys to GitHub Pages. `public/` is git-ignored and regenerated by CI -- do not commit it.

## Static Assets

- `static/files/CV-Shujie_XU.pdf` -- CV PDF (auto-synced from a separate CV repo via GitHub Actions; the `auto-update` commits are prefixed `Update from https://github.com/Shujie-Xu/CV-Shujie-XU/commit/...`).
- `static/images/profile.png` -- portrait photo referenced by `content/_index.md` `photo.src`.

## Content Editing Patterns

- Homepage copy (bio, highlights, current-work entries, intro lead) is in `content/_index.md` front matter / body. Templates are only the frame.
- Site-wide identity (name, role tagline, footer line, social URLs, menu items) lives in `hugo.toml`. Changes there ripple to header, footer, and meta tags.
- Research page is plain Markdown with `##`/`###`/`*italic*` hierarchy -- let the Markdown carry the structure, do not reach for `<div>` or custom classes.
- The CV page needs no markdown body to function; `description` + `updated` front matter + the PDF file are enough. If the PDF filename changes, update `content/cv/_index.md` `pdf` path.
